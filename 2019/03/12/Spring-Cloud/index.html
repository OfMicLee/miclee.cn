Spring Cloud

Spring-Cloud

Spring Cloud 各组件整合

标准化整合

  1. 服务网关

    • Netflix Zuul
    • Spring Cloud Gateway
      (Spring Cloud的新项目，推荐zuul用此替换)

  2. 服务注册发现

    • Netflix Eureka
    • Spring Cloud Consul
    • Zookeeper
    • Etcd
      （各注册中心区别详见另一边博文：Spring Cloud注册中心）

  3. 服务调用

    • Spring Cloud Feign
      (声明式、模板化的HTTP客户端)

  4. 负载均衡（Client Side Load Balancer）

    • Netflix Ribbon

  5. 统一配置中心

    • Spring Cloud Config
    • Netflix Archaius
    • Ctrip Apollo
    • Disconf

高级整合

  1. 服务短路

    • Netflix Hystrix

  2. 调用链路跟踪

    • Spring Cloud Sleuth
      (日志收集工具包，封装了Dapper和log-based追踪以及Zipkin和HTrace操作，为SpringCloud应用实现了一种分布式追踪解决方案。)

  3. 安全框架

    • Spring Cloud Security
    • Apache Shiro (比 Security 更加轻量）

  4. 消息驱动

    • Spring Cloud Stream
      （Spring Cloud Stream 是一个构建消息驱动的微服务应用的框架，数据流操作开发包，封装了与Redis、RabbitMQ、Kafka等发送接收消息的操作。通过 Binder 实现的抽象中间件，可以无改动迁移到不同的消息中间件实现。）

  5. 消息总线

    • Spring Cloud Bus
      （Spring Cloud Bus整合了java的事件处理机制和消息中间件，让微服务与微服务之间也能享受应用内的事件机制。具体的消息中间件还是整合rabbitmq、kafka之类的。 服务A:Event -> Channel:Message -> 服务B:Event）
